FORAGER.bs

This module implements the BS detector for the FORAGER pipeline, designed to identify and classify unsupported or hallucinated claims generated by large language models (LLMs). It extracts atomic claims from LLM responses and evaluates their validity against a retrieved set of supporting documents.

Core Functionality:

  • Extracts atomic claims using spaCy's natural language processing (NLP) pipeline, focusing on subject-verb-object (SVO) relationships.
  • Classifies the truthfulness of claims using a Natural Language Inference (NLI) model (facebook/bart-large-mnli), categorizing claims as 'entailment', 'contradiction', or 'neutral'.
  • Provides a final judgment of each claim as 'Supported', 'Contradicted', or 'Unsupported' using both NLI classification and optional fallback to vector similarity scoring.

Intended Usage: This module is intended to operate post-answer generation, verifying the factual alignment of LLM outputs to grounding documents. It plays a critical role in the FORAGER Prompt Lock Loop (PLL) by filtering out hallucinated information and increasing user trust in AI-generated responses.

Dependencies:

  • Hugging Face Transformers (facebook/bart-large-mnli)
  • spaCy (en_core_web_sm)
  • Sentence Transformers (for vector similarity fallback)

 39def clean_subtree(subtree):
 40    # Remove parentheticals from sentences
 41    cleaned = []
 42    inside_parens = 0
 43    for t in subtree:
 44        if t.text == "(":
 45            inside_parens += 1
 46            continue
 47        elif t.text == ")":
 48            inside_parens -= 1
 49            continue
 50        if inside_parens == 0 and t.text.lower() not in DISCOURSE_MARKERS:
 51            cleaned.append(t.text)
 52    return " ".join(cleaned)

 54def extract_atomic_claims(text):
 55    """
 56    This method extracts individual, atomic claims from a passage, using a pretrained spaCy language model to do so.
 57
 58    Args:
 59        text (str): The text from which to extract claims.
 60    
 61    Returns:
 62        claims (list):  A list of atomic claims from the text.
 63    """
 64    doc = nlp(text) # Convert text to annnotated spaCy doc
 65    claims = [] # Initialize empty list to hold all claims in text
 66
 67    # doc.sents is an iterator over the sentences in a processed document
 68    for sent in doc.sents:
 69
 70        # Skip any non-assertive statements by looking for hedging words
 71        if any(modal in sent.text.lower() for modal in ["might", "could", "if", "possibly", "maybe"]):
 72            continue
 73
 74        subject, verb, object_ = None, None, None
 75        passive_subject, agent = None, None
 76
 77        # Iterate through each token in the sentence
 78        for token in sent:
 79            # Identify the nominal subject
 80            if token.dep_ == "nsubj":
 81                # Grab entire phrase representing subject, not just the subject itself
 82                subject = clean_subtree(token.subtree)
 83
 84            # Identify nominal subjects in the passive voice
 85            elif token.dep_ == "nsubjpass":
 86                passive_subject = clean_subtree(token.subtree)
 87
 88            elif token.dep_ == "ROOT":
 89                verb_parts = []
 90                
 91                # Include auxiliaries like "is" and "was"
 92                for child in token.children:
 93                    if child.dep_ in ("aux", "auxpass"):
 94                        verb_parts.append(child.lemma_)
 95                
 96                # Add main verb
 97                verb_parts.append(token.lemma_)
 98
 99                # Include prepositional phrase (e.g., "in Cupertino")
100                for child in token.children:
101                    if child.dep_ == "prep":
102                        verb_parts.append(child.lemma_)
103                        for obj in child.children:
104                            if obj.dep_ == "pobj":
105                                verb_parts.append(obj.lemma_)
106                verb = " ".join(verb_parts)
107                # Find "by" + its object anywhere under the ROOT token
108                # Debugging statements
109                # print("\n[DEBUG] Verb subtree:")
110                for descendant in token.subtree: # look at all the children of the ROOT (verb)
111                    # print(descendant.text, descendant.dep_, descendant.head.text)
112                    if descendant.dep_ in ("prep", "agent") and descendant.text.lower() == "by":
113                        for child in descendant.children: # Look at all the grandchildren of the ROOT (verb)
114                            if child.dep_ == "pobj":
115                                agent = clean_subtree(child.subtree)
116            if token.dep_ == "xcomp":
117                object_ = clean_subtree(token.subtree)
118            elif token.dep_ in ("dobj", "attr", "pobj") and object_ is None:
119                object_ = clean_subtree(token.subtree)
120        
121        # Debugging statements
122        print("---")
123        print("SENTENCE:", sent.text)
124        print("PASSIVE_SUBJECT:", passive_subject)
125        print("AGENT:", agent)
126        print("VERB:", verb)
127
128        # Active voice
129        if subject and verb and object_:
130            claim = f"{subject} {verb} {object_}"
131            claims.append(claim)
132        # Passive voice -> rewrite to active
133        elif passive_subject and agent and verb:
134            claim = f"{agent} {verb} {passive_subject}"
135            claims.append(claim)
136        # Final fallback
137        elif passive_subject and verb:
138            claim = f"{passive_subject} {verb}"
139            claims.append(claim)
140
141    return claims

143def classify_claim_with_nli(claim: str, supporting_doc: str) -> str:
144    """
145    Classifies the claim based on a supporting_doc using NLI (entailment, neutral, contradiction).
146    This method uses Facebook's bart-large-mnli Natural Language Inference model based on BART architecture.
147    bart-large-mnli classifies the relationship between two pieces of text (a hypothesis and a premise) into one of three categories.
148
149    Args:
150        claim (str): The LLM-generated claim to be evaluated. The NLI uses this as the hypothesis.
151        supporting_doc (str): The retrieved context the LLM based its answer on. The NLI uses this as the premise.
152
153    Returns:
154        The degree to which the claim is supported by the document.
155
156            - "entailment": The claim is supported by the document
157            - "contradiction": The claim is directly contradicted by the document
158            - "neutral": The claim is neither supported nor contradicted by the document
159    """
160
161    input_pair = f"{claim} </s> {supporting_doc}"
162    result = nli(input_pair)[0]
163    label = result['label'].lower()     # entailment, contradiction, neutral
164    return label

166def detect_bs(embedder: FAISSEmbedder, claim: str, supporting_docs: list[str],
167              threshold_supported=0.70, threshold_contradicted=0.2) -> str:
168    """
169    Detects if a claim is Supported, Contradicted, or Unsupported by supporting_docs.
170    Prioritizes NLI (logical relation), then falls back to cosine similarity if needed.
171
172    Args:
173        claim (str): The LLM-generated claim to be inspected.
174        supporting_docs (list[str]): The documents retrieved from the knowledge base that the LLM based its claim on.
175        threshold_supported (float): The minimum vector similarity score between the claim and doc that is needed to classify a claim as supported by its docs.
176        threshold_contradicted (float): The maximum vector similarity score between the claim and doc that is needed to classify a claim as directly contradicted by its docs.
177
178    Returns:
179        The degree to which the claim is supported by the documents it used. Either Supported, Unsupported, or Contradicted.
180    """
181    # from FORAGER.embedder import FAISSEmbedder
182    from sentence_transformers import util
183    
184    # NLI Voting Phase
185    contradiction_votes = 0
186    entailment_votes = 0
187
188    if not supporting_docs:
189        return "Unsupported"
190    
191    for doc in supporting_docs:
192        label = classify_claim_with_nli(claim, doc)
193        # print(f"\n\n[DEBUG] Claim: {claim} \n| Doc: {doc} \n| NLI: \n{label}")
194        if label == "contradiction":
195            contradiction_votes += 1
196        elif label == "entailment":
197            entailment_votes += 1
198        
199    if contradiction_votes >= len(supporting_docs) / 2 + 1:
200        return "Contradicted"
201    elif entailment_votes > 0:
202        return "Supported"
203    
204    # Embedding Similarity Phase (only runs if NLI is neutral)
205    # embedder = FAISSEmbedder.create_default()
206    claim_emb = embedder.embed_text(claim)
207    support_embs = [
208        embedder.embed_text(doc["text"]) if isinstance(doc, dict) else embedder.embed_text(doc)
209        for doc in supporting_docs
210    ]
211    similarities = util.cos_sim(claim_emb, support_embs)[0]
212    max_score = similarities.max().item()
213
214    if "by definition" in claim or "refers to" in claim.lower():
215        if max_score >= 0.7:
216            return "Supported"
217    elif max_score >= threshold_supported:
218        return "Supported"
219    elif max_score <= threshold_contradicted:
220        return "Contradicted"
221    else:
222        return "Unsupported"
